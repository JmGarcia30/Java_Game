Anime Fighter Game - Complete Function Explanations
====================================================

AnimeFighter Class
==================

Properties
----------
name - Stores the fighter's name (String)
health - Current health points of the fighter (int)
stamina - Current stamina points, used for skills (int, max 120)
maxHealth - Maximum health for the fighter, used for healing limits (int)
bandageUsed - Tracks if bandage has been used this round (boolean)
potionUsed - Tracks if stamina potion has been used this round (boolean)
isStunned - NEW! Tracks if fighter is stunned and will skip their turn (boolean)
hasShield - NEW! Tracks if fighter has an active shield for damage reduction (boolean)
burnDamage - NEW! Counter for burn effect duration (int, 0 = no burn, >0 = burning turns remaining)
isSlowed - NEW! Tracks if fighter is slowed with reduced stamina gain (boolean)


Constructor
-----------
AnimeFighter(String name, int health, int stamina)
Creates a new fighter with the given name, health, and stamina values
Sets maxHealth equal to initial health for tracking purposes
Initializes both items (bandage and potion) as unused (false)
NEW! Initializes all status effect flags to false/0 (isStunned, hasShield, burnDamage, isSlowed)
Prints a message announcing the fighter entering the arena


Combat Functions
----------------

void basicAttack(AnimeFighter target)
Performs a basic attack on the target fighter
Deals 15 damage to the target (or 7-8 damage if target has shield)
NEW! Checks if target has SHIELD active - if yes, damage is reduced by 50% and shield is consumed
Gains 20 stamina for the attacker (capped at 120)
NEW! If attacker is SLOWED, stamina gain is reduced to only 10 instead of 20
NEW! Slow effect is removed after being applied
Checks if target is already defeated before attacking
Prevents negative health values
Announces defeat if target's health reaches 0

Item Functions
--------------

void useBandage()
Restores 50 health points to the fighter
Can only be used once per round
Checks if bandage was already used this round
Caps health at maxHealth (cannot overheal)
Sets bandageUsed flag to true after use
Displays current health after healing

void useStaminaPotion()
Restores 50 stamina points to the fighter
Can only be used once per round
Checks if potion was already used this round
Caps stamina at 120 (maximum stamina)
Sets potionUsed flag to true after use
Displays current stamina after restoration

void resetForNextRound()
Resets fighter to full health and starting stamina (80)
Resets item usage flags (bandageUsed and potionUsed back to false)
Called between rounds in a best-of-3 match
Ensures both fighters start fresh each round


Utility Functions
-----------------

void showStats()
Displays complete fighter information in a formatted box
Shows: Name, Current Health/Max Health, Current Stamina/Max Stamina
Shows item availability status (Available or Used)
NEW! Shows all active status effects:
  - "Status: BURNING" if burnDamage > 0
  - "Status: STUNNED" if isStunned is true
  - "Status: SHIELD ACTIVE" if hasShield is true
  - "Status: SLOWED" if isSlowed is true
Used at battle start and during each player's turn

void applyStatusEffects()
NEW! Processes ongoing status effects at the start of each turn
Applies BURN damage if burnDamage counter is greater than 0
  - Deals 5 damage to the fighter
  - Decrements burn counter by 1
  - Announces when burn effect ends (counter reaches 0)
  - Prevents health from going below 0
Called automatically at the beginning of each fighter's turn
Critical for managing damage-over-time effects

NEW! Status Effect System Detailed Explanation
===============================================

STUN (isStunned)
- Effect: Fighter completely skips their turn
- Applied by: Naruto's Rasengan (50% chance), Saitama's Consecutive Normal Punches (70% chance), Gojo's Cursed Technique Lapse Blue (100% chance)
- Duration: 1 turn (automatically removed after turn is skipped)
- Strategy: High-risk high-reward, can give you a free turn to set up combos

BURN (burnDamage counter)
- Effect: Deals 5 damage at the start of the affected fighter's turn
- Applied by: Asta's Black Slash
- Duration: 2 turns (counter decrements each turn)
- Strategy: Chip damage that adds up over time, good for finishing weakened opponents

SHIELD (hasShield)
- Effect: Reduces incoming damage by 50% on next attack received
- Applied by: Luffy's Gum-Gum Pistol (self-buff)
- Duration: Until next hit (consumed after absorbing one attack)
- Strategy: Defensive option that allows aggressive play with less risk

SLOW (isSlowed)
- Effect: Reduces stamina gain from basic attacks (20 â†’ 10)
- Applied by: Goku's Kamehameha
- Duration: 1 basic attack (removed after effect triggers)
- Strategy: Slows opponent's resource generation, limiting their skill usage

FREEZE (same as Stun but thematic)
- Effect: Identical to Stun mechanically
- Applied by: Gojo's Cursed Technique Lapse Blue (guaranteed)
- Duration: 1 turn
- Strategy: Guaranteed crowd control, most reliable disable in the game


NEW! Turn Validation System
============================
Problem Solved: Previously, if a player selected an already-used item, their entire turn would be wasted
Solution: validActionChosen boolean flag with while loop

How it works:
1. At start of turn, validActionChosen is set to false
2. Loop continues until validActionChosen becomes true
3. Player makes a choice
4. System checks if choice is valid:
   - If choosing bandage (6) but bandageUsed == true: Print error, loop continues
   - If choosing potion (7) but potionUsed == true: Print error, loop continues
   - If choosing basic attack or any skill: Always valid, set validActionChosen = true
5. Try-catch block handles invalid input types (if player types letters instead of numbers)
6. Only when validActionChosen is true does the loop exit and turn ends

Result: No more wasted turns! Players can always take a meaningful action.
